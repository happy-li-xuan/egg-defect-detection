## 1. 系统技术设计

### 1.1 开发技术

本系统采用前后端分离架构，前端基于 Uni-app 框架构建跨平台应用，后端基于 Flask + YOLOv8 提供智能检测服务。整体技术栈如下：

**前端技术栈**

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue.js | 3.x | 前端 MVVM 框架 |
| Uni-app | 3.x | 跨平台应用框架（H5 / 小程序 / App） |
| HTML5 / CSS3 | — | 页面结构与样式 |
| ES6+ | — | 前端脚本语言 |

**后端技术栈**

| 技术 | 版本 | 用途 |
|------|------|------|
| Python | 3.12 | 后端开发语言 |
| Flask | ≥3.0 | Web 框架，提供 RESTful API |
| Flask-CORS | ≥4.0 | 跨域资源共享支持 |
| Ultralytics YOLOv8 | ≥8.0 | 目标检测深度学习模型 |
| OpenCV | ≥4.8 | 图像预处理与后处理 |
| Pillow | ≥10.0 | 图像文件处理 |
| NumPy | ≥1.24 | 数值计算 |
| Gunicorn | ≥22.0 | WSGI 服务器（生产部署） |

**AI 对话模块**

| 技术 | 用途 |
|------|------|
| DeepSeek API | 大语言模型对话接口（兼容 OpenAI 格式） |
| SSE 流式输出 | 实现打字机效果实时回答 |

**部署平台**

| 平台 | 用途 |
|------|------|
| Render | 后端 API 云部署（新加坡节点） |
| GitHub | 代码版本管理与 CI/CD |

## 2. 系统架构设计

### 2.1 架构概述

本系统采用 **前后端分离** 的 B/S 架构。前端使用 Uni-app 框架开发，可编译为 H5 网页、微信小程序或移动 App；后端使用 Flask 提供 RESTful API，核心检测能力由 YOLOv8 深度学习模型提供。前后端通过 HTTP/HTTPS 进行数据通信。

**架构分层**

1. **表现层**（前端）— Uni-app 页面，负责用户交互与数据展示
2. **服务层**（后端 API）— Flask 路由处理，接收请求并响应
3. **AI 引擎层** — YOLOv8 目标检测模型 + DeepSeek 大语言模型
4. **数据层** — 图片文件、检测记录（本地存储）

### 3.3 模块详细说明

#### 3.3.1 用户认证模块
- **登录页面**（pages/auth/login.vue）：用户输入凭证进行登录
- **权限校验**：控制未登录用户对检测功能的访问

#### 3.3.2 首页资讯模块
- **检测意义文章**：展示鸡蛋缺陷检测的行业意义与背景
- **检测分类知识**：按缺陷类型（破损、裂纹、污渍等）分类展示检测知识
- **文章详情**：点击文章卡片跳转至详细内容页

#### 3.3.3 鸡蛋检测模块（核心模块）
这是系统的核心功能，流程如下：

\\\mermaid
sequenceDiagram
    actor User as 用户
    participant FE as 前端检测页面
    participant BE as Flask 后端
    participant YOLO as YOLOv8 模型

    User->>FE: 点击选择图片
    FE->>User: 打开系统相册/相机
    User->>FE: 选择鸡蛋图片
    FE->>FE: 显示图片预览
    User->>FE: 点击"开始检测"
    FE->>FE: 显示检测中状态
    FE->>BE: POST /api/detect (上传图片)
    BE->>YOLO: yolo.predict(image)
    YOLO-->>BE: 返回检测结果（类别+坐标+置信度）
    BE->>BE: 绘制标注框
    BE-->>FE: 返回 JSON（结果文本 + 标注图片 + 目标列表）
    FE->>FE: 展示检测结论
    FE->>FE: 展示标注图片
    FE->>FE: 展示检测记录
\\\"

#### 3.3.4 社区管理模块
- 文章列表展示与分类筛选
- 文章详情阅读

#### 3.3.5 消息中心模块
- 系统消息通知列表
- 消息详情查看

#### 3.3.6 服务中心模块
- **常见问题 FAQ**：可展开/折叠的问答列表
- **检测知识速览**：按主题组织的检测知识（光源、角度、阈值等）
- **AI 智能问答**：基于 DeepSeek 大语言模型的对话功能，支持流式输出

#### 3.3.7 后端 API 模块

| 接口 | 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|------|
| /api/detect | POST | file（图片文件）, conf（置信度阈值） | JSON（检测结果 + 标注图） | 鸡蛋图片缺陷检测 |
| /api/health | GET | 无 | JSON（服务状态 + 模型信息） | 健康检查 |
| / | GET | 无 | HTML | Web 版检测界面 |

后端检测流程：
1. 接收前端上传的图片文件
2. 转换为 OpenCV 格式进行预处理
3. 加载 YOLOv8 模型进行推理预测
4. 解析检测到的目标（类别、边界框、置信度）
5. 根据检测结果判断缺陷等级（合格 / 轻度污损 / 重度破损）
6. 在原图上绘制标注框并编码为 Base64
7. 返回结构化 JSON 响应

### 4.1 开发环境

| 环境 | 配置 |

### 4.2 快速启动（本地开发）

**Windows**
\\\powershell
powershell -ExecutionPolicy Bypass -File setup.ps1
\\\

**Mac / Linux**
\\\ash
chmod +x setup.sh && ./setup.sh
\\\"

### 4.3 一键配置脚本说明

\setup.ps1\（Windows）和 \setup.sh\（Mac/Linux）会自动完成：

1. 创建 Python 虚拟环境
2. 安装 \server/requirements.txt\ 中的全部依赖
3. 从 \.env.template\ 复制生成 \.env\ 环境配置文件
4. 交互式配置 AI API Key 和服务地址
5. 验证 YOLO 模型文件是否存在

### 4.4 后端启动

\\\ash
cd server
python app.py
\\\"

默认监听 \http://localhost:5000\，通过环境变量 \PORT\ 可自定义端口。

### 4.5 生产部署（Render）

本项目配置了 \ender.yaml\ 蓝图画部署文件，支持一键部署到 Render 云平台。

- **构建命令**：\pip install -r server/requirements.txt\
- **启动命令**：\gunicorn server.app:app --bind 0.0.0.0:\ --workers 1 --timeout 120\
- **运行时**：Python 3.12（通过 \untime.txt\ 锁定）
- **节点**：新加坡（\singapore\），国内访问延迟较低

### 4.7 配置优先级

前端配置读取优先级（高 → 低）：
1. 函数调用时传入的 \options\ 临时参数
2. \uni.getStorageSync()\ 运行时存储（不修改源码）
3. \common/appConfig.js\ 中的默认值

这一设计使系统在部署后仍可通过用户侧设置动态调整 API 地址和 Key，无需重新编译发布。
